# 文笔审稿 · 镇狱之渊

把一章放进第 1–10 回里，读起来是不是同一支笔。是，过；不是，标出哪一维掉了。

## 唯一标准

**`docs/standards/文笔范文标准.md` 是文笔标准本体，头十章（第 1–10 回）是范文。**
打分前先读它（尤其 §一文笔七维表、§二范文 A–J、§四反范文六种垮法）。
配套：`docs/standards/rubric.md`（节奏硬门 22%/47%/68%/89%）、`docs/standards/审查流程.md`（9 条硬门）。

## 两种模式

| 模式 | 何时用 | 输入 | 回写 frontmatter？ |
|------|--------|------|------|
| **草稿质检** | 写文章途中，写完一段/一节就检 | 一段草稿，可无 frontmatter | **否**（只口头报问题+下一步改法） |
| **定稿评分** | 一章写完，要上线/归档 | 完整一章 | **是**（写 review:+score:N/14） |

两种都对照同一标准（`docs/standards/文笔范文标准.md` + 头十章）。草稿质检是定稿评分的快照子集——同样的七维和门，只是不打最终分、不落地，让你边写边纠。

---

## 模式 A · 草稿质检（写文章途中）

目标：在还能改的时候，一秒看出这段跑没跑标准。**写一段就检一段，别攒到全章写完才发现声线跑了三节。**

### 跑什么（快检清单，按命中即报）

1. **机器地板**（最先跑，确定性卡死）：`python engine/prose_lint.py <草稿文件>`。出 ERROR（中英混写/逗号碎句/微碎片率>42%/平均段长<3.5）→ 先修这个，别往下。
2. **七维快读**（只标"有掉分风险"的维，不必逐维打分）：对 §一表过一遍，哪维像 0/1 就点出来——尤其维 2 形容词糊弄、维 3 心理直说、维 4 没有可量锚点。
3. **门禁快检**（写作中最易漏的四门，对照 `docs/standards/审查流程.md`）：
   - **硬线**：露骨 / 自伤细节 / 林窈(13)涉暧昧——命中立即停，这段必删改。
   - **声音**：林夙极轻平、苏挽客气疏离、裴无咎不说沉话、叶观澜极轻像说远事、牛阿大碎探——跑一个就标出来。
   - **反范文六红旗**（§四）：「且…且」长从句 / 逗号碎句 / 填充描写 / 单段≥5 个「——」 / 中英混写 / 形容词糊弄。
   - **钩子**（若这段是章末）：画面钩或信息钩二选一，落物象；总结式/议论式/双钩并存→标出。

### 输出（草稿质检不打最终分）

简短口头报，**不写 frontmatter**：

```
草稿质检 · <段落/小节定位>
- lint: PASS / ERROR(n) ——若 ERROR 贴出 lint 行
- 风险维: 维N(掉到几)——引这段里的真句，说差在哪
- 门: 硬线/声音/红旗/钩子 哪几条亮红——引真句
- 下一步: 一处最该改的——指明哪句、怎么改回哪条范文(A–J)
```

**只报最该改的一处**（最高优先级：硬线 > lint ERROR > 声线跑调 > 掉到 0 的维），别一次甩十条让写作中断。修完接着写，下一段写完再检。

### 写→检→改循环

```
写一段 → 模式A快检 → 有红旗? 改这一处 : 接着写下一段
            └────────────── 全章写完 → 转模式 B 定稿评分 ──────────┘
```

> 草稿质检不替代定稿评分。它是地基检查，过了只代表"这段没垮"；够不够晋江档、能不能落 score，仍要全章写完走模式 B。

## 模式 B · 定稿评分（一章写完）

### 1. 读

- 读目标章正文（frontmatter `# 第…回` 以下全文）。
- 读 `docs/standards/文笔范文标准.md`。第一次审稿这一步不可跳。
- 拿不准任何一维，翻回 `seasons/01-xianxia/chronicle/00{1..10}-*.md` 对一遍范文形状。

### 2. 七维打分（每维 0/1/2，满分 14）

逐句问：在写动作物象，还是在下形容词糊弄？按 §一表给分：

| 维 | 0 | 1 | 2（对应范文） |
|----|---|---|------|
| 1 句子节奏 | 逗号碎句 / 一个调 | 长短交替偶尔散 | 长铺短砸、一字成句（F） |
| 2 用词 | 形容词糊弄 | 动词带物象 | 名词+动作写情绪（B/D） |
| 3 潜台词 | 心理直说 | 留白但要硬猜 | 说 A 读到 B 不点破（E/H） |
| 4 感官 | "她很紧张" | 一个身体反应 | 锚定可触可量部位/温度/重量（D） |
| 5 对话 | 英文标签/谁说看不清 | 中文标签声线分得开 | 不靠标签认得谁说（E/G） |
| 6 视角 | 视角飘/上帝插嘴 | 锁单一视角 | 只写视角人能感知，判断带其色（H） |
| 7 克制 | 又解释又总结 | 收得住 | 该停就停，重量留读者（F/H） |

**< 9 分（文笔层）或任一维 0 → 退回重写**，在 `修复方向` 里写明回炉点。

### 3. 写 review 块

格式严格照已有章节（见 `001-退婚书.md`、`450-有人.md`）：

```yaml
review: |
  维度N<名>(分): 论据——引正文原句，点名对照范文哪一条（F/H/D…），说明到位或差在哪。
  …（只写有话可说的维，通常 4–6 条；满分维与失分维都要点到，失分维必写为什么）
  修复方向: 一条可执行的改法——指出具体句子/位置，给出怎么改（加一个可量锚点、瘦身某长句、删某句内心解释）。
score: N/14
```

要求：
- **论据必须引正文里的真句**，不空夸（"节奏好"无效，"开篇'归隐谷——/一道山谷，不长。'留白起，对照范文F"才有效）。
- **每条点名范文编号**（范文 A–J），把这一维钉在标准上。
- `score` 是**七维总和**（即使 review 块只列了 5 条，分数也按全部 7 维算）。
- 含引号时 `review: |` 已是 block scalar，安全。

### 4. 回写

把 `review:` 块和 `score:` 行插进该章 frontmatter，位置紧跟 `hook:` 之后、闭合 `---` 之前。
若该章已有旧 review，整块替换。**不动正文，不动其它 frontmatter 字段。**

## 反范文红旗（命中即扣维 + 写进修复方向）

对照 §四，这六种一眼机器味，命中直接压分：
1. 「且…且…」三连长从句（→ 维 1/2/3）— 最像 AI
2. 逗号碎句（微碎片率>42% / 平均段长<3.5）（→ 维 1）
3. 填充描写（屋里安静/心里咚一下）（→ 维 2/4）
4. 单段 ≥5 个「——」（→ 维 1）
5. 中英混写标签（→ 维 5，硬退）
6. 形容词糊弄（很愤怒/眼神冰冷）（→ 维 2）

可跑 `python engine/prose_lint.py 章节路径` 拿机器地板分佐证（lint 是地板，本审稿是天花板）。

## 最终判据

> 把这一章放进第 1–10 回里读，是不是同一支笔。是，过；不是，`修复方向` 写清从哪改回去。
