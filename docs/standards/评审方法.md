# 章节评审方法（手工 · 资深晋江作家视角）

> 适用：单章精修 / 单章审稿 / 主编 review / 大 sweep 子任务
> 不适用：批量化自动打分（那是 `engine/prose_lint.py` 的事，那是地板）
>
> 与既有标准的关系：
> - **文笔七维**定义在 `文笔范文标准.md` §一，本文讲**怎么手搓应用**
> - **9 硬门**定义在 `审查流程.md` §0–§9，本文讲**怎么手搓核验**
> - **章级评分表**（钩子/爽痛/反差...）在 `rubric.md`，本文不重复其条目
> - **锚点系统**引用 `handbook/AUDIT-META-05-ANCHORS.md`，**7 维趋势**引用 `handbook/AUDIT-META-01-PROSE-7DIM.md`，**9 门违规全表**引用 `handbook/AUDIT-META-03-9GATES.md`

---

## 〇、原则（不可妥协）

1. **手搓**——资深晋江作家视角，每一条 dim/gate 都靠人工读、人工对照范文 A–J、人工判定 PASS/PARTIAL/FAIL。不靠任何打分脚本、不靠"我感觉"、不靠 lint 输出替代。
2. **逐条 cite**——每个 dim 至少引一处具体行号 + 至少一处对照（范文 A–J 之一 / 模板名 / 角色声线指纹 / 锚点类型）。无 cite 的判定无效。
3. **不替代机器门**——`engine/prose_lint.py` 只作为地板（不 ERROR）。过了 lint 不代表本章合格；本章合格不代表过了 lint。
4. **承认约定**——项目级约定（如开篇钩允许旁白放大一次）应在 `CLAUDE.md` 落地后引用，不要每章重判。
5. **不替 project 背锅**——把"项目级问题"算作"本章问题"会导致评分通胀。例：ships 字段全章都臃肿，本章不单判 FAIL，记项目级 P0/P1 备忘。

## 一、文笔七维手搓指南（每维 0/1/2，总分 /14）

| 维 | 0（垮）信号 | 1（擦边）信号 | 2（范文）信号 | 范文对照 |
|----|------------|--------------|--------------|----------|
| **1 句子节奏** | 逗号碎句 / 长短一个调 | 长短交替偶尔散 | 关键处一字/单句成段（句号断镜） | 范文 A / C / F「行。」 |
| **2 用词** | 形容词糊弄（很静/咚一下） | 动词带具体物象 | 用名词和动作说情绪，不下形容词 | 范文 B / D |
| **3 潜台词** | 把心理直说 | 留白但要硬猜 | 说 A 读到 B，不点破 | 范文 E / H |
| **4 感官具体** | "她很紧张" | 写一个身体反应 | 锚定可触可量的部位/温度/重量 | 范文 D |
| **5 对话** | 英文标签 / 角色声线雷同 | 中文标签、声线分得开 | 不靠标签也认得谁在说，话里有没说的 | 范文 E / G |
| **6 视角** | 视角飘、上帝插嘴 | 锁单一视角 | 只写视角人能感知到的，判断都带他的色 | 范文 H |
| **7 克制** | 又解释又总结 | 收得住 | 该停就停，把重量留给读者 | 范文 F / H |

**手搓硬要求**：
- 每维判定必须能在章节里找到至少一处**具体行号引用**作为依据。
- 不允许"dim3 我给 1 因为潜台词偏弱"这种无 cite 的判定。
- 同一章出现两个相互矛盾的"dim7 克制满分"与"行内解释"时，dim7 严判。

## 二、9 硬门手搓核验（每门 PASS / PARTIAL / FAIL）

| 门 | 名 | PASS | PARTIAL | FAIL（须给修复建议）|
|----|----|------|---------|--------------------|
| **1** | 硬线 | 无露骨 / 无自伤 / 林窈(13) 不涉及暧昧 | — | 任一违例 → 整批打回 |
| **2** | 锚点 | 6 级升温曲线无跳级，ships 字段已注锚点类型 | ships 字段 500–1000 字符 / 锚点说明塞进正文行内 | 一章连跳两级（追溯上一章） |
| **3** | 前情钩 | 画面钩 / 信息钩二选一，落物象 | 双钩并存但互洽（单钩复合）| 总结式 / 议论式 / 纯进度总结 / 真双钩（画面+议论并列）|
| **4** | 缺席 | cast 与近 15 章错开节奏；配角 ≤10 章；阿湄 ≤3 章 | cast 连 3 章同 / 列 cast 但未出场未注 / 新角色破场后 4–5 章 | 连续 4 章 cast 同 / 配角 >10 章缺席 / 新角色破场后 ≥6 章 |
| **5** | 回改 | 现有场景加 1–3 句，标【已回改·锚点说明】| 单句 200+ 字符（违反"1–3 句"精神） | 改对话主干 / 删已有锚点 / 二次回改已被标记章节 |
| **6** | 节奏 | beat 字段 = 起/承/转/合 + in_beat/总数 | beat 编号断点（25+ 章跳）| 起用法误用于弧内章 / beat 分母错（如 ch492「20/6」）|
| **7** | 声音 | 每角色声线指纹对齐（林夙极轻平/苏挽客气疏离/林彻绕弯/叶观澜极轻远事/牛阿大碎探/...）| 1 处偶发偏离 | 林夙说快话 / 苏挽说软话 / 裴无咎说沉话 / 三人同场讲大道理 |
| **8** | 群像 | 每出场角色有专属动作 + 专属台词 + 专属情绪反应 | 1 处工具人式出场（只为递信息）| ≥2 处工具人式出场 |
| **9** | frontmatter | 8 字段完整 / 标题 2–4 字 / cast≥4 / hook `\|` block scalar | ships 500–1000 字符 / beat 字段缺分母 | cast<4 / 标题劣化（纯地点/纯人名/语气词） / hook 含引号但 inline / ships>1000 字符 |

**手搓硬要求**：
- FAIL 必须有具体修复建议（不是"建议优化"）。
- PARTIAL 须说明本章不单判 FAIL 的理由 + 项目级备忘编号。

## 三、输出格式（写入章节 frontmatter）

review 直接写入章节文件 frontmatter，紧接在 ships 字段之后、闭合 `---` 之前：

```yaml
---
season: S
chapter: N
title: 标题（2–4 字）
cast: [角色 A, 角色 B, ...]
pov: POV 角色
beat: 承/转/合·描述 N/M
ships:
  CP×CP: 本章埋的伏笔或情感张力
hook: |
  钩子（含引号时必须 | block scalar）
review: |
  维度1节奏(N): [证据 1–3 行 + 范文对照]
  维度2用词(N): [...]
  维度3潜台词(N): [...]
  维度4感官(N): [...]
  维度5对话(N): [...]
  维度6视角(N): [...]
  维度7克制(N): [...]
  9硬门: 门1 PASS｜门2 PASS｜门3 PARTIAL｜...｜门9 FAIL
  修复方向：
  1. [项目级 P0/P1 备忘，本章不单判 FAIL]
  ...
score: N/14
---

# 正文
```

**YAML 硬约束**：
- `review:` 必须是 `|` block scalar（多行 YAML）——章节多处含引号，inline 会破坏解析。
- block scalar 内容相对 `review: |` 缩进 **2 空格**，且所有行缩进一致。
- `score` 必须是 `N/14` 格式（数字 + 斜杠 + 14），不接受 `13 分`、`12.5/14`、`满分` 等变体。

**9 硬门单行紧凑写法**：`门1 PASS｜门2 PASS｜门3 PARTIAL｜门4 PASS｜门5 FAIL｜门6 PASS｜门7 PASS｜门8 PASS｜门9 PARTIAL`

## 四、不该写进 review 的内容

- ❌ 机器门 lint 结果摘要（那是 lint 报告的事）
- ❌ "整体感觉不错 / 不错" 之类无 cite 总评
- ❌ 不指明行号的"潜台词强" "用词准"
- ❌ 模板饱和判定（"是...那种久"模板用了 5+ 次）但未点出本章是否触发该模板
- ❌ 把"项目级问题"算作"本章问题"（参见 §〇-5）
- ❌ 未经 `handbook/AUDIT-META-05-ANCHORS.md` 验证就断言"本章升了几级"

## 五、范文对照速查（每 dim 必须能引到至少一处）

| 范文 | 章节 | 标准 |
|------|------|------|
| A 开篇强度 | ch001 | 第一句直接抛冲突，不铺背景 |
| B 重生冷叙 | ch001 | 用身体感觉勾时间 |
| C 动作经济 | ch001 | 短句切镜，先给结果再补原因 |
| D 感官锚点 | ch001 / ch003 | 可触可量，部位+温度+重量 |
| E 对话潜台词 | ch001 / ch003 | 说的是 A，读到的是 B |
| F 字的重量 | ch002 / ch003 / ch010 | 关键一两个字单拎成句成段 |
| G 群像独特 | ch002 | 配角有专属动作 + 专属台词 |
| H 通篇潜台词 | ch010 | 自洽的隐喻系统，全章不解释 |
| I 声音温度 | ch001 | 声音 + 比喻 + 隐含动因 |
| J 结尾意象 | ch003 / ch010 | 落物的意象，不总结 |

## 七、worked example · ch001《退婚书》

> 维度1节奏(2):「签了它，你还是林家人；不签，你今晚就从祠堂里抬出去。」「这冷他熟。」「死寂。」「不是他停的。是林夙抬指，轻轻一弹。」「这桩婚，不是你们退我。/是我，退你们。」「嗯。」——单句成段+句号断镜贯穿全章；对照范文 A/C/F「行。」式留白。
> 维度2用词(2):「这冷他熟」「膝盖下的寒气一寸寸往上爬」「车帘掀了一指宽」「手心攥得发紧」——全章无「很」字起头；用身体感觉勾时间是范文 B/D 原型。
> 维度3潜台词(2):「等我哪天死了，你别来给我烧纸」「这一世，会相识的。」「像在看一件该处理掉的旧物」「薄得像刀背的杀气」「让他再长长」——五条线全是说 A 读到 B。
> 维度4感官(2):触·袖角擦手背（范文 D 金句）+ 触·指尖在袖里压一下（范文 D 金句）+ 视·车帘掀一指宽（可量）+ 声·低半度（范文 I 金句）+ 温·手炉拢怀里。
> 维度5对话(2):林彻「——」截断+绕弯+反问；林夙单字短句；苏挽客气疏离+反问；叶观澜极轻像说远事——不靠标签也认得。
> 维度6视角(1):主体锁林夙 POV，但「祠堂角落...他叫牛阿大」段 + 末段叶观澜 POV 段两处旁白放宽——严判扣分；属开篇钩结构必要。
> 维度7克制(1):原版范文 F/J 双金句满分；但 round-2 在 body line 84 加「是...那种久」长句（5+「且」嵌套 / 单句 200+ 字符）——是该模板 ch361–410 复用 27 次的源头。
> 9硬门: 门1 PASS｜门2 PASS｜门3 PASS（单钩复合）｜门4 PARTIAL（赤渊列 cast 未出场）｜门5 PARTIAL（line 84 超「1–3 句」精神）｜门6 PASS｜门7 PASS｜门8 PASS｜门9 PARTIAL（ships 1000+ 字符）
> 修复方向: ships 字段瘦身迁 `docs/anchors/` / body line 84 回改原版 / 视角放宽约定落地 CLAUDE.md / 钩子粗判校准
> **score: 12/14**

## 九、本方法的边界（什么时候不适用）

- **批量化自动打分**——超过 5 章的连续 sweep 应先写脚本扫**候选问题**，再人工逐章判定。脚本输出仅作为"该章可能在哪几门有问题"的提示，不替代手搓。
- **机器门 floor**——`engine/prose_lint.py` 仍是地板，本章 ERROR 直接退回，不进入 review 流程。
- **跨卷/跨季 review**——本章方法只看单章 + frontmatter，不替代 `handbook/PHASE*-*` 报告的跨段审计。
