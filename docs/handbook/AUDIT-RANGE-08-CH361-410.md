## 典型崩坏案例

### 案例 1：ch361《换不了的》—— 锚点说明占据正文 40%

line 90（锚点说明本体）：
> 「凌朔往林夙那边那一眼——比叶观澜刚出去时，停得久了一点，是林夙把"他没答、且他袖里有东西、且他不打算说"这件事用不答替代说出来、且她要把"她不问、且她不问是他自己留着"这件事从"看"的久里压进心里、且不必她先开口说"你留着"的那种久。」

138 字符一句。本章共 99 行，这种"且……且……且……"的锚点说明出现 3 次。同时该章 frontmatter 的 `ships` 字段里还有 600+ 字符的【已回改·锚点说明】。等于"读者要看两遍同一件事的解释"。

### 案例 2：ch383《读出来》—— OFFSCREEN stretch 内最大违规

林夙 POV，亲手用复盘之眼读余伯命格。三行字浮出来：「魂印可転，命数可改」「此法非死人復生」「用法者项若枢。立契者林崇」——这是全书最大的真相点之一，按 OFFSCREEN 设计应通过别人的转述或动作发生，**不应由林夙本人当场读到**。

按 v4 规则 ch303-449 是「林夙 OFFSCREEN，所有人以为他死了」区间，ch383 的「林夙亲自在场读出立契者名字」直接打破了 OFFSCREEN 的核心张力——读者早就知道他还活着，OFFSCREEN 区间没有 OFFSCREEN 化。

### 案例 3：ch391《认出来》—— 章节功能 = "林夙见林崇"

按"假死"叙事结构，林夙应在 ch449 之前都不与林崇见面（见面即破 OFFSCREEN 假象）。ch391 林夙带裴无咎、凌朔、阿湄四人**公开到槐树底下见林崇**，林崇亲口说"你上辈子我也卖了你"。这是 OFFSCREEN 假象的彻底崩裂。

但奇怪的是，ch397 叶观澜 POV 章的 beat 标的是「假死后·叶观澜以为得手 1/2」——意味着 OFFSCREEN 假死的"启动点"被写在了 ch391 之后，但 ch391-396 林夙仍在全场推进。**OFFSCREEN 框架内部逻辑不连贯**。

### 案例 4：ch407《知道了》—— 全范围最佳

POV 苏挽，beat 标"合·三季决战前夕 24/24"，是苏挽×阿湄双人场的关键章。苏挽说出「他，上一世也是这样」——把 OFFSCREEN stretch 里苏挽要说的最重要的话接住。阿湄没有接，只是往她那边看过来——这是全书最重的双人场景之一。

但——按 OFFSCREEN 规则苏挽不应该知道「他上一世」是被卖出去的（这是林夙自己的前世记忆），她没有理由说出这句话。除非设定改写：苏挽前世结局自己也"重生"过一次。这是 v4 OFFSCREEN 规则的另一处不一致。

## 核心结论

**本范围 20 章样本的真正问题，不是文笔，是结构。**

文笔七维均分 9.15（接近晋江档底线），rubric 均分 8.85（半数低于 9 分门槛），节奏硬门仅 89% 钩子命中——但这些都属可改范畴。

**最大问题**：v4 OFFSCREEN 规则（ch303-449 林夙 OFFSCREEN）在 ch361-410 这 50 章里完全未落地。80% 章节林夙 POV、全章与他人同框、主动推进主线、使用核心能力（复盘之眼）、与林崇当面会面。**唯一 OFFSCREEN 化的 ch397 内部逻辑自洽（叶观澜"以为得手"），但与前后章的衔接断裂。**

按官方 OFFSCREEN 规则本范围应整批打回重写——但打回意味着要重写 50 章，等于把 OFFSCREEN stretch 的核心叙事重做。这不是 review agent 能给出的建议，这是项目层的架构决策。

**建议**：
1. **架构层**：OFFSCREEN 规则与当前章节状态二选一。要么修订 v4 把范围缩到 ch397 前后，要么批量改写 ch361-396 让林夙真正 OFFSCREEN。
2. **元数据层**：把所有 `ships` 字段里的【已回改·锚点说明】迁到 `docs/anchors/` 目录，frontmatter 只留指针。
3. **锚点池**：「视·目光/留 且…且…且…那种久」模板进冷却池，本范围已使用 27 次，稀缺性归零。
4. **行文层**：所有"心里把这件事压了一遍"+"在心里把 X 替代说出来"的解释性补叙砍到每章 ≤ 2 处。
